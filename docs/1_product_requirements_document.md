# 버스타볼까 (Bus Tabolka) PRD
버전: v1.0 | 작성일: 2026-01-09

## 1. 개요
- **한 줄 요약**: 나만의 맞춤형 환승 추천과 탑승 이력/메모를 기록하는 개인 맞춤 버스 앱
- **해결하는 문제**: 기존 앱의 부족한 개인화(맞춤 추천 미비)와 탑승 이력/메모 기능 부재 해결
- **목표 일정**: 개인 프로젝트로 일정에 쫓기지 않고 완성도 있게 개발 (Web Initial -> App Later)

## 2. 사용자 정의
- **타겟 사용자**: 본인 (개발자 자신)
- **사용 맥락**: 출퇴근, 낯선 곳 이동, 주말 약속 계획 등 복합적 상황
- **피해야 할 것**: 환승/탑승 이력 미표시, 단편적인 정보 제공

## 3. 핵심 기능
| ID | 기능명 | 설명 | 사용자 가치 | MVP 포함 |
| :--- | :--- | :--- | :--- | :--- |
| FEAT-1 | 맞춤 환승 추천 | 짐 유무, 날씨 등 상황에 맞춰 덜 걷거나 편한 경로 추천 | 상황별 최적 이동 | ✅ (1순위) |
| FEAT-2 | 탑승 히스토리 | 내가 탄 버스와 경로를 자동/수동으로 기록 | 이동 패턴 분석, 재사용 | ✅ (2순위) |
| FEAT-3 | 노선 메모장 | 특정 버스 노선이나 정류장에 대한 개인적인 메모 기록 | 경험 기반 정보 축적 | ✅ |

## 4. User Flow
```mermaid
graph TD
    A[앱 진입] --> B{로그인 상태?}
    B -->|No| C[로그인/회원가입]
    C --> B
    B -->|Yes| D[메인 (검색/즐겨찾기)]
    
    D --> E[경로 검색]
    E --> F{조건 설정}
    F -->|짐 많음/비 옴| G[최소 도보/환승 추천 목록]
    F -->|일반| H[최단 시간 추천 목록]
    
    G & H --> I[상세 경로 보기]
    I --> J[탑승 시작 (기록)]
    J --> K[이동 중]
    K --> L[도착 완료]
    L --> M[히스토리 저장 & 메모 작성]
```

## 5. MVP 범위 (10줄 캡슐)
- **목표**: 내가 실제로 쓸 수 있는 수준의 맞춤 경로 추천 및 기록 웹 서비스
- **타겟 사용자**: 1인 (개발자 본인)
- **핵심 기능**: 카카오맵 기반 경로 검색 + 커스텀 필터링(맞춤 추천) + 이동 기록 DB 저장
- **성공 지표**: 실제 출퇴근/이동 시 기존 앱 대신 이 앱을 사용함
- **포함**: 이메일 로그인, 경로 검색, 즐겨찾기, 탑승 기록, 노선 메모
- **제외 (v2)**: 앱 스토어 출시(PWA로 우선 대응), 복잡한 소셜 기능
- **기술 스택**: Next.js, Supabase, Kakao Map API, 공공데이터포털 API
- **검증 방법**: 실제 대중교통 이용 시 테스트

## 6. 성공 지표
| 지표 유형 | 지표명 | 목표값 |
| :--- | :--- | :--- |
| 노스스타 | 실사용 빈도 | 주 5회 이상 (출퇴근 등) |
| 입력 지표 | 히스토리/메모 수 | 누적 50건 이상 기록 |

## 7. 범위 외 (Non-goals)
- 친구 추가 / 공유 기능
- 버스 외 교통수단(택시, 자전거 등) 통합 예약
